## 🏗️ 系统架构

### 技术栈
- **前端**：React 18 + Ant Design 5 + Vite
- **后端**：Spring Boot 3 + MyBatis-Plus
- **数据库**：MySQL 8.0
- **部署平台**：
  - 前端：Vercel（Serverless）
  - 后端：Railway（容器化）
  - 数据库：Railway MySQL

### 架构图
```
用户浏览器
    ↓
Vercel CDN（前端静态资源）
    ↓
Railway 后端服务（Spring Boot）
    ↓
Railway MySQL（数据持久化）
```

## 🚨 故障排查

### 常见问题及解决方案

#### 1. 前端页面无法访问
**现象**：打开域名显示 404 或无法连接

**排查步骤**：
1. 检查 Vercel 控制台部署状态
2. 检查域名配置是否正确
3. 检查 GitHub 仓库是否有更新

**解决方案**：
- 重新部署：Vercel 控制台 → Redeploy
- 检查环境变量：VITE_API_URL 是否正确

#### 2. API 接口返回 500 错误
**现象**：前端显示"加载数据失败"

**排查步骤**：
1. 查看 Railway 后端日志
2. 检查数据库连接是否正常
3. 检查数据库表结构是否完整

**解决方案**：
- 检查环境变量：DATABASE_URL 是否正确
- 重启后端服务
- 检查数据库表结构

#### 3. 数据库连接失败
**现象**：后端日志显示数据库连接错误

**排查步骤**：
1. 检查 MySQL 服务状态
2. 检查连接信息是否正确
3. 检查网络连接

**解决方案**：
- 在 Railway 控制台重启 MySQL 服务
- 检查环境变量配置
- 检查防火墙设置

#### 4. 数据丢失或损坏
**现象**：数据异常或丢失

**解决方案**：
1. 立即停止服务，防止数据进一步损坏
2. 从备份恢复数据
3. 检查代码是否有 bug

## 📝 更新部署流程

### 代码更新步骤

1. **本地修改代码**
   ```bash
   # 修改代码后
   git add .
   git commit -m "更新说明"
   ```

2. **推送到 GitHub**
   ```bash
   git push origin master
   ```

3. **自动部署**
   - Railway 自动拉取最新代码并部署后端
   - Vercel 自动拉取最新代码并部署前端
   - 等待 2-3 分钟部署完成

4. **验证部署**
   - 访问前端页面测试功能
   - 检查后端 API 是否正常

### 数据库变更

如果需要修改数据库结构：

1. **在 Railway MySQL 中执行 SQL**
   ```sql
   -- 例如添加新字段
   ALTER TABLE activities ADD COLUMN new_field VARCHAR(100);
   ```

2. **更新实体类代码**
   - 修改对应的 Entity 类
   - 添加 getter/setter 方法

3. **提交代码并部署**
   - 推送到 GitHub
   - 等待自动部署

## 🔐 安全注意事项

1. **环境变量**
   - 数据库密码等敏感信息保存在 Railway 环境变量中
   - 不要硬编码在代码中
   - 不要将 .env 文件提交到 GitHub

2. **数据库安全**
   - 定期更换数据库密码
   - 限制数据库访问 IP
   - 定期备份数据

3. **代码安全**
   - 定期更新依赖包
   - 检查安全漏洞
   - 使用 HTTPS 访问
