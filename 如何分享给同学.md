# 龙山青年共创实验室 - 分享指南

## 方法一：让同学本地运行（推荐，最稳定）

### 准备工作

你需要给同学提供以下文件：
1. `longshan-app` 整个文件夹（可以压缩成 zip）
2. 或者只提供前端打包文件 + 说明

### 同学需要安装的环境

1. **MySQL 8.0** - 数据库
2. **Node.js 18+** - 运行前端
3. **Java 17+** - 运行后端
4. **Maven 3.9+** - 构建后端

### 快速启动步骤

#### 方式 A：一键启动（最简单）

1. 双击运行 `start.bat`
2. 等待服务启动（约 20 秒）
3. 浏览器访问 http://localhost:3000

#### 方式 B：手动启动

**步骤 1：启动 MySQL**
- 确保 MySQL 服务正在运行
- 确保 root 密码是 `root123`

**步骤 2：初始化数据库**
```bash
mysql -u root -proot123 longshan_db < database/init_clean.sql
```

**步骤 3：启动后端**
```bash
cd backend
mvn spring-boot:run
```
等待显示 "Started LongshanApplication"

**步骤 4：启动前端**
```bash
cd frontend
npm install  # 第一次需要安装依赖
npm run dev
```

**步骤 5：访问**
浏览器打开 http://localhost:3000

## 方法二：部署到 Vercel（需要你有账号）

### 前端部署

1. 注册 Vercel 账号：https://vercel.com
2. 安装 Vercel CLI：`npm i -g vercel`
3. 登录：`vercel login`
4. 部署前端：
   ```bash
   cd frontend
   vercel --prod
   ```

### 后端部署

后端需要部署到支持 Java 的平台，如：
- Railway
- Render
- Heroku

## 方法三：使用 ngrok（临时演示）

如果你只是临时给同学演示，可以使用 ngrok：

1. 注册 ngrok：https://ngrok.com
2. 下载并配置 authtoken
3. 启动你的本地服务
4. 运行：`ngrok http 3000`
5. 把生成的 https 链接发给同学

## 常见问题

### Q1: 同学说 "无法连接数据库"
- 检查 MySQL 是否启动
- 检查密码是否正确（默认 root123）
- 检查数据库 `longshan_db` 是否存在

### Q2: 同学说 "端口被占用"
- 修改 `backend/src/main/resources/application.yml` 中的端口
- 修改 `frontend/vite.config.js` 中的端口

### Q3: 同学说 "npm install 很慢"
- 使用淘宝镜像：`npm config set registry https://registry.npmmirror.com`

### Q4: 如何修改后端 API 地址？
- 修改 `frontend/src/services/api.js` 中的 baseURL
