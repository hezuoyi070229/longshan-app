# 部署到云服务器指南

## 方案：使用 Docker 部署（推荐）

### 1. 服务器准备

需要一台云服务器（阿里云/腾讯云/AWS等）：
- 配置：2核4G 以上
- 系统：Ubuntu 20.04/22.04
- 开放端口：80, 8080

### 2. 安装 Docker

```bash
# 安装 Docker
curl -fsSL https://get.docker.com | sh

# 启动 Docker
sudo systemctl start docker
sudo systemctl enable docker
```

### 3. 部署 MySQL

```bash
# 运行 MySQL 容器
docker run -d \
  --name mysql \
  -e MYSQL_ROOT_PASSWORD=root123 \
  -e MYSQL_DATABASE=longshan_db \
  -p 3306:3306 \
  mysql:8.0

# 导入数据
docker cp database/init_clean.sql mysql:/tmp/
docker exec -i mysql mysql -uroot -proot123 longshan_db < /tmp/init_clean.sql
```

### 4. 部署后端

创建 `backend/Dockerfile`：

```dockerfile
FROM openjdk:17-jdk-slim
COPY target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "/app.jar"]
```

构建并运行：
```bash
# 构建后端
cd backend
mvn clean package -DskipTests

# 构建 Docker 镜像
docker build -t longshan-backend .

# 运行后端容器
docker run -d \
  --name backend \
  --link mysql:mysql \
  -p 8080:8080 \
  -e SPRING_DATASOURCE_URL=jdbc:mysql://mysql:3306/longshan_db \
  longshan-backend
```

### 5. 部署前端

创建 `frontend/Dockerfile`：

```dockerfile
FROM node:18-alpine as builder
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
```

创建 `frontend/nginx.conf`：

```nginx
server {
    listen 80;
    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
    location /api {
        proxy_pass http://backend:8080;
    }
}
```

构建并运行：
```bash
cd frontend
docker build -t longshan-frontend .
docker run -d \
  --name frontend \
  --link backend:backend \
  -p 80:80 \
  longshan-frontend
```

### 6. 访问

浏览器访问服务器 IP 地址即可。

## 快速部署脚本

创建 `deploy.sh`：

```bash
#!/bin/bash

# 停止并删除旧容器
docker stop mysql backend frontend 2>/dev/null
docker rm mysql backend frontend 2>/dev/null

# 启动 MySQL
docker run -d --name mysql \
  -e MYSQL_ROOT_PASSWORD=root123 \
  -e MYSQL_DATABASE=longshan_db \
  -v $(pwd)/database/init_clean.sql:/docker-entrypoint-initdb.d/init.sql \
  mysql:8.0

# 等待 MySQL 启动
sleep 30

# 构建并启动后端
cd backend
mvn clean package -DskipTests
docker build -t longshan-backend .
docker run -d --name backend --link mysql:mysql -p 8080:8080 \
  -e SPRING_DATASOURCE_URL=jdbc:mysql://mysql:3306/longshan_db \
  longshan-backend

# 构建并启动前端
cd ../frontend
docker build -t longshan-frontend .
docker run -d --name frontend --link backend:backend -p 80:80 longshan-frontend

echo "部署完成！访问 http://服务器IP"
```

## 无服务器方案：使用 Vercel + Railway

### 前端部署到 Vercel

1. 推送代码到 GitHub
2. 在 Vercel 导入项目
3. 自动部署，获得域名如 `longshan.vercel.app`

### 后端部署到 Railway

1. 注册 Railway 账号
2. 从 GitHub 部署后端
3. 添加 MySQL 插件
4. 获得后端域名如 `longshan-api.up.railway.app`

### 配置前端 API 地址

修改 `frontend/src/services/api.js`：
```javascript
const api = axios.create({
  baseURL: 'https://longshan-api.up.railway.app/api',
  // ...
})
```

---

## 推荐

- **有服务器**：用 Docker 方案，最稳定
- **无服务器**：用 Vercel + Railway，免费且稳定
