# 云端部署详细指南

## 方案概述

使用 **Vercel（前端）+ Railway（后端+数据库）**，完全免费且稳定。

```
用户浏览器 → Vercel（前端）→ Railway（后端）→ Railway MySQL（数据库）
```

---

## 第一步：准备 GitHub 仓库

### 1.1 创建 GitHub 仓库

1. 访问 https://github.com/new
2. 仓库名：`longshan-app`
3. 选择 **Public**（公开）
4. 点击 **Create repository**

### 1.2 上传代码

在 `longshan-app` 文件夹中执行：

```bash
# 初始化 git
git init

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit"

# 连接远程仓库（替换为你的用户名）
git remote add origin https://github.com/你的用户名/longshan-app.git

# 推送
git push -u origin main
```

---

## 第二步：部署后端到 Railway

### 2.1 注册 Railway

1. 访问 https://railway.app
2. 点击 **Login**，选择 **Continue with GitHub**
3. 授权 Railway 访问你的 GitHub

### 2.2 创建后端项目

1. 点击 **New Project**
2. 选择 **Deploy from GitHub repo**
3. 选择你的 `longshan-app` 仓库
4. 点击 **Add Variables**，添加以下环境变量：

```
DATABASE_URL=mysql://root:password@mysql.railway.internal:3306/railway
DB_USERNAME=root
DB_PASSWORD=你的数据库密码（自动生成）
```

5. 点击 **Deploy**，等待部署完成
6. 部署完成后，点击 **Settings** → **Domains**
7. 复制域名，例如：`https://longshan-api.up.railway.app`

### 2.3 添加 MySQL 数据库

1. 在项目页面点击 **New**
2. 选择 **Database** → **Add MySQL**
3. 等待 MySQL 创建完成
4. 点击 MySQL 服务，查看 **Connect** 信息
5. 更新后端的环境变量为 MySQL 的连接信息

### 2.4 初始化数据库

在 Railway 的 MySQL 服务中：

1. 点击 **Query** 标签
2. 复制 `database/init_clean.sql` 的内容
3. 粘贴并执行，创建表和数据

---

## 第三步：部署前端到 Vercel

### 3.1 注册 Vercel

1. 访问 https://vercel.com
2. 点击 **Sign Up**，选择 **Continue with GitHub**
3. 授权 Vercel 访问你的 GitHub

### 3.2 创建前端项目

1. 点击 **Add New Project**
2. 选择你的 `longshan-app` 仓库
3. 配置如下：
   - **Framework Preset**: Vite
   - **Root Directory**: `frontend`
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`

4. 点击 **Environment Variables**，添加：

```
VITE_API_URL=https://你的后端域名.up.railway.app/api
```

5. 点击 **Deploy**，等待部署完成

### 3.3 获取前端域名

部署完成后，Vercel 会提供一个域名，例如：
`https://longshan-app.vercel.app`

---

## 第四步：配置 CORS（跨域）

后端需要允许前端域名访问。修改 `backend/src/main/java/com/longshan/config/WebConfig.java`：

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/**")
                .allowedOrigins(
                    "http://localhost:3000",
                    "https://你的前端域名.vercel.app"  // 添加这行
                )
                .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
                .allowedHeaders("*")
                .maxAge(3600);
    }
}
```

然后重新推送到 GitHub，Railway 会自动重新部署。

## 第五步：验证部署

## 常见问题

### Q1: Railway 部署失败
**解决**：查看日志，通常是数据库连接问题，检查环境变量是否正确。

### Q2: 前端无法连接后端
**解决**：
1. 检查 `VITE_API_URL` 是否正确
2. 检查后端的 CORS 配置是否包含前端域名
3. 检查后端是否正常运行

### Q3: 数据库数据丢失
**解决**：Railway 的免费 MySQL 在重启后数据会保留，但如果删除服务会丢失。重要数据请定期备份。
